XCSS
a wayward framework based on css

Introduction

XCSS is a framework for webapp building that is totally different from frameworks like Angular and React. It is an experiment on how to do web development using css as a framework.
Since all css rules act on selections of elements, expressing the logic with css rules should be a powerfull way to implement functionality in web pages. Adding the xcss.js script to a web page should be enough to enable the new css functionality of xcss.

Development of XCSS.js began with an experiment to implement the extends keyword for css. I thought is it was easier to make it possible to use keywords in css ruls then learning sass, Alsso the extends keyword means nothing in a css selector unless you would create an element by that name. 


    red-button {
      background-color:red;
    }

    red-green-button extends dialog{
       border:2px solid green;
    }

The next step was that i realised that i could use the content property in css rules to allow templating. 


    div[string-content] {
      content:"'<span>hello</span>'"
    }

    div[url-content] {
      content:url(page2.html)
    }

Then i thought of a way to do routing in css. By introducing the ONCLICK and more ON event keywords that allowed adding event listeners to elements that changed the location hash and a WHEN keyword than applied a style depending on the location hash, i created a simple routing mechanism. 


    button ONCLICK showpage2 {
    }

    div WHEN showpage2 {
      content:url(page2.html)
    }
